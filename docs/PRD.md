# 마이링크 (MyLink) - 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 사용자가 흩어져 있는 여러 개의 링크(SNS, 포트폴리오, 쇼핑몰, 블로그 등)를 하나의 웹페이지로 모아서 쉽게 공유할 수 있도록 지원하는 멀티 링크 서비스.
- **대상 사용자**: 
  - 여러 채널을 운영하며 팬들과 소통하는 **크리에이터 및 인플루언서**
  - 포트폴리오, 이력서, 개인 연락처를 깔끔하게 공유하고자 하는 **프리랜서 및 구직자**
  - 여러 상품 페이지나 예약 링크를 홍보해야 하는 **소상공인 및 비즈니스 운영자**

## 2. 주요 기술 스택 (Tech Stack)
- **프론트엔드 UI/디자인 시스템**: **shadcn/ui**를 활용하여 일관되고 세련된 모던 웹 컴포넌트 구성
- **백엔드 및 인프라**: **Firebase** (Authentication, Firestore Database, Storage 등 활용)

## 3. 핵심 기능 목록

### 필수 기능 (MVP - 초기 배포 목표)
1. **사용자 인증**: **구글 소셜 로그인** (Firebase Auth 활용) 및 로그아웃
2. **프로필 관리**: 고유 URL(ID) 설정, 프로필 이미지 업로드, 닉네임 및 소개말(Bio) 작성
3. **링크 관리**: 링크 추가, 수정, 삭제 (사용자당 평균 4~5개의 링크 등록을 가정)
4. **링크 파비콘(Favicon)**: 링크 등록 시, 해당 URL의 파비콘 이미지를 추출하여 링크 아이템 아이콘으로 표시
5. **퍼블릭 페이지 (뷰어)**: 모바일 친화적으로 렌더링된 사용자 고유 페이지(`서비스도메인.com/사용자ID`)
6. **기본 테마**: 라이트/다크 모드 등 가장 기본적인 2~3가지 테마 템플릿 제공

### 선택/확장 기능 (추후 고도화 목표)
1. **링크 클릭 수 조회**: 각 링크별 클릭 조회수(Click-through) 트래킹 기능
2. **추가 소셜 로그인**: 구글 외에 카카오, 애플 등 다른 소셜 로그인 연동 확장
3. **커스터마이징 및 꾸미기**: 사용자 지정 배경(단색, 그라데이션, 이미지), 폰트 스타일, 버튼 모양(둥근 정도 등) 변경
4. **소셜 아이콘 배치**: 메인 링크와 별도로 화면 하단이나 상단에 작은 아이콘(인스타그램, 유튜브, X 등)만 배치하는 기능

*※ 참고: 본 서비스는 현재 유료 과금 모델 없이 모든 기능을 무료로 제공하는 것을 전제로 합니다.*

## 4. 각 기능의 상세 설명

### 4.1 사용자 인증
- **Firebase Authentication**을 이용하여 **구글 소셜 로그인**을 단일 인증 방식으로 제공합니다.
- 복잡한 이메일 가입 절차 없이 클릭 한 번으로 빠른 서비스 진입을 유도합니다.

### 4.2 프로필 관리
- **고유 URL 생성**: 최초 로그인 후 영문/숫자 조합의 고유 ID를 설정합니다. 중복 확인이 필수적입니다.
- **프로필 이미지**: 이미지를 업로드하고 원형으로 크롭되어 표시되도록 합니다. (Firebase Storage에 저장)
- **소개글**: 최대 150~200자 이내의 간단한 자기소개를 작성할 수 있습니다.

### 4.3 링크 관리
- **추가/수정/삭제**: 사용자는 표시할 텍스트(Title)와 실제 연결될 주소(URL)를 입력하여 링크 블록을 생성합니다. (평균 4~5개 내외의 링크 등록을 상정하며, 순서 변경이나 비활성화 토글 기능은 제외합니다.)
- **파비콘 적용**: 사용자가 URL을 입력하면, 해당 웹페이지의 파비콘(Favicon) 이미지를 자동으로 추출하여 링크 블록 내의 썸네일 아이콘으로 적용합니다.

### 4.4 퍼블릭 페이지
- 사용자가 설정한 정보들을 바탕으로 최종 뷰어 페이지가 렌더링됩니다.
- shadcn/ui 기반의 깔끔한 카드 형태로 프로필 이미지, 소개글, 링크들이 표시됩니다.
- 모바일 화면 비율에 최적화된 UI를 기본으로 하며, PC에서 접속하더라도 가운데 정렬된 모바일 형태의 뷰를 제공합니다.
